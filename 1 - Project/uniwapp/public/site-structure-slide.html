Uniwapp Architecture Overview

Site Structure For Management Presentation

One-slide map of the guest journey, reception journey, and core testing points.

Entry Point: index.html (Cruise Overview)
Shared Assets: style.css + logo.png + localStorage state

Guest Module Flow

  • index.htmlGuest landing and day selector
  • day.html?day=1..8Daily details: excursions and onboard activities
  • signup.htmlBooking login and excursion selection
  • my-tours.htmlFinal review of submitted selections

Main nav links connect Cruise Overview, Select My Tours, and My Excursions.

Reception Module Flow

  • reception.htmlRoom submission dashboard
  • reception-room.htmlRoom details and submitted selections
  • reception-guest-excursions.htmlExcursions grouped by guest
  • reception-totals.htmlTotals per excursion and event

Users switch from Guest to Reception via the topbar module button.

Backend/Data Layer

  • server.js (API and data orchestration)
  • data/excursions.csv (catalog of tours and events)
  • data/guests.csv (booking and guest identity)
  • data/submissions.json (stored choices and status)

Testing Checklist

  • Navigation test: all links and day transitions work
  • Booking test: valid and invalid login scenarios
  • Selection test: per-guest limits and confirmations
  • Reception test: room status, guest grouping, totals accuracy
